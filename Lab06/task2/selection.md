# Pattern Selection

| Scenario | Family | Pattern | Why |
|----------|--------|---------|-----|
| **01** App Config | Creatinal | **Singleton** | One instance globally accessible |
| **02** Payment Gateway | Structural | **Adapter** | Translate modern <-> legacy interfaces |
| **03** Notifications | Behavioral | **Observer** | One-to-many dependency |
| **04** Report Builder | Creational | **Builder** | Same steps, different formats |
| **05** Undo/Redo | Behavioral | **Command** | Encapsulate actions for reversal |
| **06** 50k Particles | Structural | **Flyweight** | Share intrinsic state |

** References:** Ch. 6 (families) + Ch. 7 (specific patterns)
