# Task 2: Pattern Selection - "The Right Tool"

## Methodology
1. Read each scenario carefully, identifying the core problem
2. Map the problem to a pattern family using Ch. 6 definitions:
- Need to control creation ? -> Creational
- Need to compose/adapt structure? -> Structural
- Need to manage communication? -> Behavioral
3. Select the specific pattern from the family
4. Justify with book references
5. Implement two scenarios with complete, runnable code

## Files
- `selection.md` - Pattern selection for all 6 scenarios
- `decision_journal.md` - Decision journal with rationale
- `impl_scenario_XX.js` - Implemented scenarios (2 files)
